Advanced Data System Lab Intern
2019.10 - present
— Performance modeling and optimization of distritbuted ML —
Show More
▶

Guided by Prof. Cheng Li

We aim at building a universal and accurate performance modeling framework for data parallel distributed deep neural networks on three mainly used gradient aggregation topology. Furthermore, based on the model auto optimization of hardware configuration of the system could be executed.

  • Profiled and analyzed the computation & communication time of data parallel distributed DNN models, e.g. vgg13, using Horovod (A distributed ML training framework) profiler, Apache MXNet (A deep learning framework) profiler and Chrome tracing tool.

  • Proposed general formulas of communication time of three mainly used gradient aggregation topology: parameter server, reduction ring and tree.

  • Measured message transmission bandwidth with Nvidia collective communication library(NCCL) test tool and profile OpenMPI allreduce operation using Nvidia nvprof tool.

  • Built and visualized our performance model based on abstraction that training iteration of distributed DNN can be formulated as a DAG mapped onto timeline, and the DAG consists of data I/O operator, computationtime and communication operator of each layer of DNN.

  • Our performance model achieves less than 5% error rate on scalability prediction and iteration time prediction with experiments on vgg13 and resnet.

The name of my research project is "wearable pain measurement" (which is an independent research project).We focus on developing automated ways to measure pain responses, which can be performed on patients anytime, even on patients in coma or outside the hospital.

Here are what we have done:

  • Designed and implemented the prototype of a wearable pain measurement system, which includes two key components:
    I have designed the wearable device and pain stimuli mechanism using Fusion 360 (a Autodesk product) and implemented the device using 3D printer, laser cutter.
    1. A wearable device that is able to give subject proper, controllable level of pain stimuli then collect certain skin signals for later analysis.

    2. A skin signal analysis program which is able to do signal processing and statistic analysis, and output the relationship between skin signal pattern with the pain stimuli level.

  • Designed the pain stimuli mechanism and the wearable device using Fusion 360 (A mechanical design software) then implemented it using 3D printer and laser cutter, etc.

  • Built the circuit and coded on Arduino for device control and data communication, used sensors to collect skin galvanic response signals and for vertical pressure calibration.

  • Wrote program for signal processing and statistic analysis like t-test and clustering with python.

  • Did some pilot user study.
